Vor der Einstellung des Web-Clusters sollte die entsprechende Bitrix Dokumentation sorgfältig gelesen werden. Die Einstellung kann dann so vorgenommen werden, dass die Leistungsfähigkeit und die Fehlertoleranz der Website wesentlich erhöht werden.

Man sollte lieber nicht alle Möglichkeiten des Web-Clusters (Web-Cluster Optionen) sofort einsetzen. Da Web-Cluster ein ziemlich kompliziertes System ist, sollten die Einstellungen dieses diesem Systems allmählich und schrittweise erfolgen: Bitrix Web Cluster.

  1. Im memcached den Daten-Cache einstellen, dabei eine optimale Cache-Größe auswählen (beispielsweise 128 Mb auf jedem gestarteten memcached Server).
  2. Einen zweiten Anwendungsserver einstellen, Content-Synchronisierung mithilfe von csync (oder mithilfe einer anderen Technologie) einstellen.
  3. An den zweiten sowie an alle anderen Anwendungsserver ein Überwachungssystem anbinden.
  4. Lastverteilung und SSL-Terminierung (falls erforderlich) einstellen, Sitzungen zwischen Anwendungsservern verteilen: a. Allgemeine Sitzungen, welche in den Datenbanken gespeichert werden, und b. Nutzersitzungen, welche abhängig von IP-Adressen verteilt und separat gespeichert werden.
  5. SQL-Replikation einstellen. Bei Bedarf vertikale Fragmentierung (Sharding) benutzen.
  • Stellen Sie sicher, dass mit der ausgewählten Konfiguration der Web-Cluster-Architektur die gesetzten Ziele (also Verfügbarkeit und/oder Leistungsfähigkeit) erreicht werden. Überprüfen Sie dabei folgende wichtige Schwerpunkte: Eine richtige Anzahl von Anwendungsservern, Arbeitsmodus der Lastverteilung, Arbeitsmodus der Replikation und die Anzahl der Datenbankserver etc. Zusätzlich können Spezifikationen im Miniformat erstellt werden, in denen die Architektur des Web-Clusters mit Rücksicht auf Anforderungen einer konkreten Website beschrieben wird.
